Características de MongoDB

  • Modelo de datos orientado a documentos: Los datos se almacenan en documentos BSON (una extensión de JSON).
    Cada documento puede tener una estructura diferente, lo que proporciona gran flexibilidad
  • No requiere esquema fijo (schema-less): Puedes guardar documentos con diferentes estructuras dentro de la misma colección.
  • Escalabilidad horizontal: Soporta sharding, una técnica para distribuir datos entre múltiples servidores. Esto permite escalar horizontalmente grandes volúmenes de datos
  • Alta disponibilidad y replicación: Usa replica sets, que son grupos de servidores MongoDB que mantienen las mismas copias de datos.
    Si el nodo principal falla, otro toma su lugar automáticamente (failover)
  • Consultas potentes: Permite realizar consultas complejas, incluyendo filtros, rangos, búsquedas de texto, agregaciones y más.
    Cuenta con el framework de agregación, una poderosa herramienta para procesar y transformar datos.
  • Índices: Puedes crear índices en cualquier campo para acelerar las consultas.
    Soporta índices compuestos, geoespaciales, de texto, entre otros.
  • Alto rendimiento: Rápido en lectura y escritura, especialmente en cargas de trabajo no estructuradas o semi-estructuradas.
    Compatibilidad con múltiples lenguajes: Drivers oficiales para JavaScript (Node.js), Python, Java, C#, PHP, Ruby, Go, etc.
  • Interfaz amigable: Ofrece herramientas como MongoDB Compass (interfaz gráfica) y la Mongo Shell para trabajar con la base de datos.
  • Base de datos orientada a la nube: Tiene su servicio en la nube llamado MongoDB Atlas, que facilita la administración, escalado y monitoreo automático

Operaciones básicas en MongoDB

Operación Descripción Sentencia MySQL
CREATE Inserta un documento en la colección db.collection.insertOne({campo: valor})
READ Busca documentos que coincidan con la condición db.collection.find({campo: valor})
UPDATE Actualiza un documento que cumple el filtro db.collection.updateOne({filtro}, {$set: {campo: nuevo_valor}})
DELETE Elimina un documento que cumple la condición db.collection.deleteOne({campo: valor})
